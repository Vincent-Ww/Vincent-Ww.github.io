leetcode 哈希表


哈希表

1.两数之和

给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。
你可以假设每种输入只会对应一个答案。但是，数组中同一个元素不能使用两遍。

示例:
*给定 nums = [2, 7, 11, 15], target = 9

因为 nums[0] + nums[1] = 2 + 7 = 9
所以返回 [0, 1] *

构建一个哈希表，键值对为（index数字对应的另一个数字大小，index）

建立哈希表和查找哈希表放在一次循环里，先查找在放入哈希表，可以免去检查两个index是不是相等。


1.三数之和

给你一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a，b，c ，使得 a + b + c = 0 ？请你找出所有满足条件且不重复的三元组。
注意：答案中不可以包含重复的三元组。

示例：
给定数组 nums = [-1, 0, 1, 2, -1, -4]，
满足要求的三元组集合为：
[[-1, 0, 1],
[-1, -1, 2]]

这是我的解法，思路和两数之和相似，都是先建立一个哈希表，然后查找。

重点是不能又重复，实现方式：先把数组排序，哈希表里每个元素的index是排好序的数组里这个数最大的index。比如排好序是-1，-1，0，1，2，4. 那么hashtable[-1]=1而不是0。 然后让匹配它的两个数都小于他。

核心思想是：保证a<b<c。哈希表里的数是c。

接下来考虑如何a,b不重复， 当通过两层for循环遍历所有a,b可能性时，第一点是j从i+1开始（这个简单），第二点是：如果这次遍历到的a和上一次一样（i.e. nums[i]=nums[i-1），就跳过。同理这次遍历到的b和上次一样的话(i.e. nums[j]==nums[j-1])，也跳过(continue)。

第二种解法：双指针

这里second和third构成双指针。 在排好序的数组里，用双指针找出 一对和为特定值得数 得复杂度是O(N)。

个人觉得双指针这样写不明显，下面是另一种写法。
